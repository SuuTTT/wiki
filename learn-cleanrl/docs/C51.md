# Categorical DQN (C51): From First Principles

This tutorial builds C51 from scratch. We start from the Reinforcement Learning basics, define every symbol, then derive the C51 update rule and loss.

## 1. Start From the RL Problem

We model the environment as a Markov Decision Process (MDP):

- State: $s \in \mathcal{S}$
- Action: $a \in \mathcal{A}$
- Reward at time $t$: $R_t$
- Discount factor: $\gamma \in [0, 1)$
- Transition rule: $P(s_{t+1} \mid s_t, a_t)$

An agent chooses actions to maximize long-term reward.

### Return

The discounted return from time $t$ is

$$
G_t = \sum_{k=0}^{\infty} \gamma^k R_{t+k+1}
$$

Because rewards and transitions are random, $G_t$ is random.

## 2. What Classic DQN Learns

For action values, classic DQN learns the expectation of return:

$$
Q(s,a) = \mathbb{E}[G_t \mid s_t=s, a_t=a]
$$

So each action gets one number: its mean return.

Why this can be limiting:

- Action A: 50 percent chance of +10, 50 percent chance of -10, mean = 0
- Action B: always 0, mean = 0

Same mean, very different risk profile.

## 3. Distributional RL Idea

Instead of only learning the mean, learn the full distribution of return.

Define

$$
Z(s,a) := G_t \mid (s_t=s, a_t=a)
$$

Here $Z(s,a)$ is a random variable, not a scalar.

The normal Q-value is recovered as the expectation:

$$
Q(s,a) = \mathbb{E}[Z(s,a)]
$$

So Distributional RL contains classic RL as a special case.

## 4. Why It Is Called C51

C51 represents the return distribution on a fixed discrete support (atoms).

Choose:

- Minimum return: $V_{\min}$
- Maximum return: $V_{\max}$
- Number of atoms: $N$ (C51 uses $N=51$)

Atom locations are fixed:

$$
z_i = V_{\min} + i\Delta z, \quad i=0,\dots,N-1,
$$

with spacing

$$
\Delta z = \frac{V_{\max}-V_{\min}}{N-1}.
$$

For each action, the network predicts probabilities over these atoms:

$$
p_i(s,a) \approx \Pr(Z(s,a)=z_i), \quad \sum_i p_i(s,a)=1.
$$

The expected Q-value used for action selection is

$$
Q(s,a) = \sum_{i=0}^{N-1} z_i\,p_i(s,a).
$$

## 5. Distributional Bellman Target

For one transition $(s,a,r,s',d)$, where $d$ is done flag:

1. Select next action using expected values (Double DQN style):

$$
a^* = \arg\max_{a'} \sum_i z_i\,p_i(s',a').
$$

2. Take the next-state distribution for $a^*$ and apply reward and discount:

$$
T Z = r + \gamma(1-d) Z(s',a^*).
$$

If terminal ($d=1$), this becomes a point mass at $r$.

## 6. Why Projection Is Needed

After shift-and-scale, target atom values are

$$
\hat z_j = \mathrm{clip}\big(r + \gamma(1-d) z_j,\; V_{\min},\; V_{\max}\big).
$$

These $\hat z_j$ usually do not land exactly on fixed support points $z_i$.
So we project probability mass back to nearest bins.

For each source atom $j$ with probability $p_j(s',a^*)$:

$$
b_j = \frac{\hat z_j - V_{\min}}{\Delta z}, \quad
l_j = \lfloor b_j \rfloor, \quad
u_j = \lceil b_j \rceil.
$$

Then add mass to projected target distribution $m$:

- If $l_j = u_j$: put all mass in bin $l_j$.
- Else:

$$
m_{l_j} \mathrel{+}= p_j(s',a^*)(u_j - b_j),
$$

$$
m_{u_j} \mathrel{+}= p_j(s',a^*)(b_j - l_j).
$$

This preserves total probability and gives target distribution $m$ on fixed atoms.

## 7. Cross-Entropy Loss From First Principles

Now we have:

- Target distribution: $m$ (from projection)
- Predicted distribution for sampled action: $p(s,a)$

### What Is Cross-Entropy?

For discrete distributions $m$ and $p$:

$$
H(m,p) = -\sum_i m_i \log p_i.
$$

Interpretation:

- If $p_i$ is high where target $m_i$ is high, loss is small.
- If model assigns tiny probability to target-high bins, loss is large.

Relation to KL divergence:

$$
\mathrm{KL}(m\|p)=\sum_i m_i\log\frac{m_i}{p_i}=H(m,p)-H(m).
$$

Since $H(m)$ does not depend on model parameters, minimizing cross-entropy is equivalent to minimizing $\mathrm{KL}(m\|p)$.

So C51 training loss for one sample is

$$
\mathcal{L}_{\mathrm{C51}} = -\sum_{i=0}^{N-1} m_i \log p_i(s,a).
$$

In code, we usually compute log-probabilities with log-softmax for numerical stability.

## 8. Full C51 Training Step (One Minibatch)

1. Sample transitions from replay buffer.
2. Compute online network logits for current $(s,a)$ and convert to log-probabilities.
3. Compute target next-action $a^*$ using expected values from online network.
4. Get next-action distribution from target network.
5. Apply distributional Bellman shift-and-scale.
6. Project onto fixed atoms to produce $m$.
7. Compute cross-entropy loss $-\sum_i m_i\log p_i(s,a)$.
8. Backpropagate and optimize.
9. Periodically update target network.

## 9. Practical Hyperparameter Notes

- $V_{\min}, V_{\max}$ should cover typical return range; too narrow causes clipping distortion.
- $N=51$ is standard and efficient.
- Keep reward scaling and clipping consistent with your chosen support.
- Prioritized replay, dueling heads, and noisy nets combine well with C51 (as in Rainbow).

## 10. Key Takeaways

- Classic DQN learns only $\mathbb{E}[Z]$.
- C51 learns an approximation of the full return distribution $Z$.
- Bellman update is applied to distributions, then projected back to fixed atoms.
- Training uses cross-entropy between projected target distribution and predicted distribution.
- Expected Q-values are still available via $\sum_i z_i p_i$, so action selection remains simple.

## 11. Equation-to-Code Mapping (Core Segments)

This section maps each important definition/equation to the minimal PyTorch code.

### 11.1 Data structure for $Z(s,a)$

In C51, we do not store $Z(s,a)$ as one scalar. We represent it by:

- Fixed support atoms: `atoms` of shape `[N]`
- Predicted probabilities per action: `pmfs` of shape `[B, A, N]`

```python
# N atoms on fixed support [v_min, v_max]
self.register_buffer("atoms", torch.linspace(v_min, v_max, steps=n_atoms))  # [N]

logits = self.network(x).view(B, n_actions, n_atoms)  # [B, A, N]
pmfs = torch.softmax(logits, dim=2)                   # [B, A, N], sum over N = 1
```

Mathematically, `pmfs[b, a, i]` corresponds to $p_i(s_b, a)$.

### 11.2 Recover expected value $Q(s,a)=\sum_i z_i p_i$

```python
q_values = (pmfs * self.atoms).sum(dim=2)  # [B, A]
```

This is the exact expectation of the learned distribution.

### 11.3 Greedy action selection $a^*=\arg\max_{a'}Q(s',a')$

```python
# use expected values for argmax
next_q = (next_pmfs_all_actions * atoms).sum(dim=2)  # [B, A]
next_actions = torch.argmax(next_q, dim=1)           # [B]

# gather distribution for chosen action a*
next_pmfs = next_pmfs_all_actions[torch.arange(B), next_actions]  # [B, N]
```

### 11.4 Bellman shift-and-scale $\hat z_j = r + \gamma(1-d)z_j$

```python
# rewards: [B, 1], dones: [B, 1], atoms: [N]
shifted_atoms = rewards + gamma * (1 - dones) * atoms.view(1, -1)  # [B, N]
tz = shifted_atoms.clamp(v_min, v_max)                              # [B, N]
```

### 11.5 Projection to fixed support

```python
delta_z = (v_max - v_min) / (n_atoms - 1)
b = (tz - v_min) / delta_z             # [B, N]
l = b.floor().clamp(0, n_atoms - 1)
u = b.ceil().clamp(0, n_atoms - 1)

target_pmfs = torch.zeros_like(next_pmfs)  # [B, N]

d_m_l = (u + (l == u).float() - b) * next_pmfs
d_m_u = (b - l) * next_pmfs

for i in range(target_pmfs.size(0)):
	target_pmfs[i].index_add_(0, l[i].long(), d_m_l[i])
	target_pmfs[i].index_add_(0, u[i].long(), d_m_u[i])
```

This code implements:

$$
m_{l_j} \mathrel{+}= p_j(u_j-b_j), \qquad
m_{u_j} \mathrel{+}= p_j(b_j-l_j).
$$

### 11.6 Cross-entropy loss $-\sum_i m_i \log p_i(s,a)$

```python
# predicted distribution for sampled (s,a): [B, N]
_, pred_pmfs = q_network.get_action(observations, actions.flatten())

# numerically stable variant in this tutorial
loss = (-(target_pmfs * pred_pmfs.clamp(min=1e-5, max=1 - 1e-5).log()).sum(dim=1)).mean()
```

Equivalent log-softmax style (often preferred):

```python
logits = q_network.network(observations).view(B, n_actions, n_atoms)
chosen_logits = logits[torch.arange(B), actions.flatten()]          # [B, N]
log_pred_pmfs = torch.log_softmax(chosen_logits, dim=1)
loss = (-(target_pmfs * log_pred_pmfs).sum(dim=1)).mean()
```

### 11.7 Full training core (compact)

```python
data = rb.sample(batch_size)

with torch.no_grad():
	# 1) next action from expected value
	logits_next_online = q_network.network(data.next_observations).view(B, A, N)
	pmfs_next_online = torch.softmax(logits_next_online, dim=2)
	next_actions = torch.argmax((pmfs_next_online * atoms).sum(2), dim=1)

	# 2) next distribution from target net for selected actions
	logits_next_target = target_network.network(data.next_observations).view(B, A, N)
	pmfs_next_target = torch.softmax(logits_next_target, dim=2)
	next_pmfs = pmfs_next_target[torch.arange(B), next_actions]  # [B, N]

	# 3) Bellman transform + projection -> target_pmfs
	shifted_atoms = data.rewards + gamma * (1 - data.dones) * atoms.view(1, -1)
	tz = shifted_atoms.clamp(v_min, v_max)
	# ... compute b, l, u, then scatter mass into target_pmfs ...

# 4) predicted pmf for sampled actions
logits = q_network.network(data.observations).view(B, A, N)
log_pred = torch.log_softmax(logits[torch.arange(B), data.actions.flatten()], dim=1)

# 5) cross-entropy optimize
loss = (-(target_pmfs * log_pred).sum(1)).mean()
optimizer.zero_grad()
loss.backward()
optimizer.step()
```

## 12. PyTorch Grammar for Absolute Beginners

If you are new to Deep Learning code, read this section slowly. It explains the exact language mechanics behind lines like `self.network(x).view(B, n_actions, n_atoms)`.

### 12.1 Objects and method calls

In Python, if `obj` is an object, then `obj.method(arg)` means:

1. Find function `method` inside `obj`.
2. Run it with input `arg`.
3. Return the output.

So in

```python
self.network(x)
```

- `self` is the current neural-network class instance.
- `self.network` is an `nn.Sequential` model (a stack of layers).
- `x` is input tensor (batch of states).
- output is a tensor called logits.

### 12.2 What is a tensor?

A tensor is a numeric array with shape metadata.

- Scalar shape: `[]`
- Vector shape: `[N]`
- Matrix shape: `[B, D]`
- 3D tensor shape: `[B, A, N]`

For C51:

- `B` = batch size (how many samples processed together)
- `A` = number of actions
- `N` = number of atoms

### 12.3 Why `self.network(x)` gives a flat output first

The final linear layer is:

```python
nn.Linear(84, self.n * n_atoms)
```

So each sample gets a flat vector of length `A*N`.
For a whole batch, output shape is `[B, A*N]`.

### 12.4 Grammar of `.view(B, n_actions, n_atoms)`

`view` means "reinterpret the same memory with a new shape".

```python
logits = self.network(x).view(B, n_actions, n_atoms)
```

Step-by-step:

1. `self.network(x)` creates `[B, A*N]`.
2. `.view(B, A, N)` reshapes it to `[B, A, N]`.
3. No values change, only how we index them changes.

Concrete example:

- If `B=32`, `A=2`, `N=101`, then `A*N=202`.
- Before view: `[32, 202]`.
- After view: `[32, 2, 101]`.

Important rule: total number of elements must match.

$$
B\cdot(A\cdot N) = B\cdot A\cdot N
$$

### 12.5 Why `softmax(..., dim=2)`

```python
pmfs = torch.softmax(logits, dim=2)
```

`dim=2` means normalize along atom axis `N`.

- For each sample `b` and action `a`, the 101 atom scores become probabilities.
- Therefore `pmfs[b, a, :]` sums to 1.

That is exactly what a categorical distribution needs.

### 12.6 Grammar of `(pmfs * self.atoms).sum(dim=2)`

```python
q_values = (pmfs * self.atoms).sum(dim=2)
```

`self.atoms` has shape `[N]`. PyTorch broadcasts it to `[B, A, N]` automatically when multiplied with `pmfs`.

- Multiply: probability times atom value, elementwise.
- Sum over atoms (`dim=2`): expected value per action.
- Result shape: `[B, A]`.

This is the code form of

$$
Q(s,a)=\sum_i z_i p_i(s,a).
$$

### 12.7 Grammar of advanced indexing

```python
chosen_pmfs = pmfs[torch.arange(B), actions]  # [B, N]
```

Interpretation:

- `torch.arange(B)` gives sample indices `[0, 1, 2, ..., B-1]`.
- `actions` gives one action index for each sample.
- Together they pick one action distribution per sample.

So from `[B, A, N]`, you extract `[B, N]`.

### 12.8 Grammar of no-grad block

```python
with torch.no_grad():
	...
```

Inside this block, PyTorch does not build gradient graph.

- Saves memory
- Faster
- Correct for target computation (we do not optimize target network outputs here)

### 12.9 Grammar of cross-entropy line

```python
loss = (-(target_pmfs * pred_pmfs.clamp(min=1e-5, max=1 - 1e-5).log()).sum(dim=1)).mean()
```

Break it apart:

1. `pred_pmfs.clamp(...)` avoids exact 0 or 1 to prevent `log(0)`.
2. `.log()` computes $\log p_i$.
3. `target_pmfs * log_probs` gives $m_i \log p_i$ elementwise.
4. `.sum(dim=1)` sums over atoms: $\sum_i m_i\log p_i$ per sample.
5. unary minus gives cross-entropy.
6. `.mean()` averages over batch.

### 12.10 Grammar of backprop lines

```python
optimizer.zero_grad()
loss.backward()
optimizer.step()
```

- `zero_grad`: clear old gradients.
- `backward`: compute new gradients using chain rule.
- `step`: update parameters using optimizer rule (Adam here).

### 12.11 A fully expanded version of one core line

Compact form:

```python
logits = self.network(x).view(B, n_actions, n_atoms)
```

Expanded beginner form:

```python
flat_logits = self.network(x)                       # [B, A*N]
B = flat_logits.shape[0]
logits = flat_logits.view(B, self.n, self.n_atoms) # [B, A, N]
```

Both are equivalent. The compact form is just shorter.
